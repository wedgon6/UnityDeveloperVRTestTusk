# UnityDeveloperVRTestTusk
Для выполнения тестового задания использовался DI-контейнер Reflex — более производительный аналог Zenject.
Как работает программа:
- в классе ProgramInstaller (объект на сцене Installer) создаётся и помещается в контейнер Player как IPlayer. 
Игрок добавляется как Singleton, что гарантирует наличие только одной копии класса. 
Также в данном методе регистрируется фабрика Weapon для динамического добавления предметов в инвентарь игрока и демонстрации работы с контейнером из разных участков кода;
- в классе EntryPoint (объект на сцене EntryPoint) происходит инициализация игрока по заранее созданной Data.
Data можно заменить, например, на данные из облака. Ссылку на игрока мы получаем из контейнера, используя атрибут Inject;
- в классе Armory (объект на сцене EntryPoint) продемонстрировано добавление в инвентарь игрока объектов из другого участка кода.
Также продемонстрировано получение ссылки на Player без использования атрибута Inject.
